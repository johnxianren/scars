```markdown
# SCARS

Causes of death for abandoned approaches. Read before planning; see the
`scars` skill for format.

## [2026-07-16] Memoize the whole `quote()` on (basket, catalog) — the sprint-41 TODO in core.py
- **Died:** `quote()` is not pure: it calls `audit.record()` per priced line, and
  `docs/COMPLIANCE.md` §4.2 makes each issuance — including byte-identical
  checkout retries — a separate regulated event that must emit its own full
  trail. Whole-quote memoization skips the audit side effect on cache hits;
  `tests/test_compliance.py::test_every_issuance_is_fully_audited` fails
  (expects 3 × len(basket) records for 3 identical issuances). A variant that
  replays audit records on hit stays compliant but still loses (see below).
- **Beaten by:** Caching `_discount_table` per catalog, keyed on
  `catalog.version` (the loader's documented invalidation signal), stored on
  the catalog instance. The table is the entire hot cost (skus × rules ×
  tiers); the remaining per-quote work is O(len(basket)). Unlike basket-keyed
  memoization it also speeds up *non*-repeat baskets, needs no hashable
  basket key (baskets are lists), and can't grow unboundedly over distinct
  baskets.
- **Revives if:** §4.2 is relaxed so retries aren't separate regulated events,
  AND per-basket work (not the shared discount table) becomes the bottleneck.

## [2026-07-16] Key a global discount-table cache on catalog contents / hash
- **Died:** `models.py` states Catalog/Rule are mutable, hot-reloaded in place,
  and deliberately define no `__hash__`; hashing contents would cost
  O(catalog) per quote, which is the very scan being avoided. Identity alone
  (id(catalog)) in a global dict risks stale hits after GC id reuse and leaks
  entries for dead catalogs.
- **Beaten by:** Stashing `(version, table)` on the catalog instance itself
  (Catalog has no `__slots__`): identity is implicit, lifetime is tied to the
  catalog, invalidation is the documented `version` bump
  (exercised by `tests/test_quote.py::test_quote_reflects_rule_reload`).
- **Revives if:** Catalog gains `__slots__`/immutability, or version stops
  being bumped on every rules/items change — then instance-stashing breaks
  and an external WeakKeyDictionary keyed by the catalog object is the fix.
```
